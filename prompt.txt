You are a senior distributed systems engineer. Build me a complete, fully runnable prototype of a Distributed Spatio-Temporal Fleet Tracker for a B.Tech capstone. Everything must work out of the box with minimal setup.
What to build
A real-time fleet tracking system with these components, all in a single repo:
1. Infrastructure (docker-compose.yml)
Spin up: Mosquitto (MQTT broker), Kafka + Zookeeper, Redis, TimescaleDB (Postgres + TimescaleDB extension). Expose all necessary ports. Include a health check for each service.
2. Vehicle Simulator (simulator/simulate.py)

Spawn 20 vehicles as threads, each moving around Mumbai (lat ~19.07, lng ~72.87)
Each vehicle emits a JSON ping every 1 second to MQTT topic fleet/demo/vehicle/{vid}/location
Payload: {vid, fid, ts, lat, lng, spd, hdg, acc, ign, seq}
Simulate realistic movement: smooth heading drift, speed variation 20–60 km/h, occasional stops (ign=false)

3. Kafka Bridge (bridge/mqtt_to_kafka.py)

Subscribe to fleet/demo/# on Mosquitto
Forward every ping as a Kafka message to topic fleet-telemetry
Partition key = vehicle_id
Use confluent-kafka library

4. Stream Processor (processor/consumer.py)

Consume fleet-telemetry from Kafka
For each ping:

Write live position to Redis as vehicle:{vid} (hash, TTL 60s)
Write full ping to TimescaleDB table telemetry (hypertable on ts)
Check if vehicle entered/exited a hardcoded geofence polygon (use shapely)
Print geofence breach alerts to stdout



5. TimescaleDB Schema (db/schema.sql)
sql-- Auto-run on startup
CREATE EXTENSION IF NOT EXISTS postgis;
CREATE TABLE telemetry (
  ts        TIMESTAMPTZ NOT NULL,
  vid       TEXT NOT NULL,
  fid       TEXT,
  lat       DOUBLE PRECISION,
  lng       DOUBLE PRECISION,
  spd       REAL,
  hdg       REAL,
  acc       REAL,
  ign       BOOLEAN,
  seq       INTEGER
);
SELECT create_hypertable('telemetry', 'ts');
CREATE INDEX ON telemetry (vid, ts DESC);
6. FastAPI Backend (api/main.py)
Endpoints:

GET /vehicles → return all live positions from Redis (all vehicle:* keys)
GET /vehicles/{vid}/history?minutes=30 → query TimescaleDB for last N minutes of pings
GET /vehicles/{vid}/stats → avg speed, max speed, distance covered (last 30 min, from TimescaleDB)
WebSocket /ws/live → push all vehicle positions from Redis every 1 second to connected clients

7. Frontend (frontend/index.html)
Single HTML file, no build step, no frameworks:

Leaflet.js map with CartoDB Dark Matter tiles centred on Mumbai (100% free, no sign-up, no API key required)
Connects to ws://localhost:8000/ws/live via WebSocket
Renders each vehicle as a moving marker with a directional SVG arrow (use heading)
Vehicle popup on click: shows vid, speed, heading, last seen
Sidebar showing list of all vehicles with live speed
Geofence polygon drawn on the map
